PCMS: Quality Analyst Co-op

August 2018 - December 2018

As a Quality Analyst as PCMS I had the responsibility of testing the software before we delivered to the customer. My testing would vary between testing new functionality and defect fixes. When I was testing new functionality I would read through the BRD (Business Requirements Document) and develope test cases to check once it was added. For defect testing I would check to see if the original defect was fixed, and then test to make sure the fix didn't break another part of the program.

Starting as a QA I did not know what Requirement-Based Testing was yet, but I learned pretty quickly. We were trained to test all of the obvious scenarios and then told to try and break the software if we could. Breaking the software before it got delivered to the customer is always preferred as it gives the team an opportunity to go in an correct mistakes before the cutomer sees. More often than not there is some far-fetched way to break the code, it's just a matter of figuring out how. Learning RBT I believe will prepare me for developement work because I will already be thinking of ways the program could break and take that into consideration when writing the code.

After testing I would report back my results, usually just as a comment on the Jira ticket, but if I were to run into a fatal error I would bring it up right away and that would usually become the highest priority defect. Either way, communication skills are vital when testing. You need to be able to tell someone exactly what you mean and mean exactly what you say. My communication skills improved once I understood the applications better and I was able to be more efficient in testing.

In addition to RBT and communication skills, I also got to learn about other aspects of the developement process:

  • Learned Requirement-Based Testing
  • Learned how to navigate an extensive databse to access relevent information
  • Learned the Agile Software Developement process
  • Learned how to read and modify files in the program
  • Worked on a scrum team and contributed to daily scrum meetings

Word Search Mania

Word Search Mania was created using on Python 3 and the PyQt5 framework. The application will randomly pull words out from a 370K word dictionary file to populate the puzzle. The user also has the option to create a custom word bank from the customization menu.

In addition to making your own word bank, the user can select the size of word search they want as well. This can vary the difficulty of the games. Highscores are recorded and stored so users can try to beat thier previous scores and comapre with other.

This project utilized the teams object-oriented programming skills as well as external libraries. The PyQT5 library made it very easy to create and modify the GUI. I learned a lot about working as a team on a single application and how important communication is when everyone is working on a seperate part.
